# Starter Adapter

Choose `Starter` when you want the smallest version of the workflow that still behaves like a real system.

## Best fit

- solo founder;
- one primary owner;
- early-stage product;
- low parallelism;
- still wants role separation and reviewable artifacts.

## What to keep

- constitution;
- project workflow;
- traceability rules;
- PM / TL / Developer role docs;
- task, checkpoint, review, and handoff templates;
- root `AGENTS.md` discipline.

## What to avoid

- overbuilding governance;
- installing heavy parallel-work machinery before it is needed;
- making checklists so heavy that the team stops using them.

## Practical baseline

- one active stage at a time is acceptable;
- direct TL execution can still exist, but should remain exception-shaped;
- delegated execution is recommended when the work is non-trivial.
