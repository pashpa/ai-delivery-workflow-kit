# Examples

This folder is intentionally lightweight.

The workflow kit is meant to be installed into real repositories, not operated from examples alone.

## Fast install example

Use [`../QUICKSTART.md`](../QUICKSTART.md) first.

The expected user behavior is:

1. give the workflow kit URL to Codex App or another coding agent;
2. give the agent the target product repo;
3. ask it to install the workflow and stop before product implementation.

## Example scenarios to model in a target repo

- new product from zero;
- existing repo with a new feature;
- existing repo with a bug investigation;
- cross-surface feature contract;
- small Task vs full Lane package.

Even in small examples, keep separate role contexts for PM, TL, Developer, and QA work when the work is non-trivial.
