# PM Onboarding

Status: Active

This document helps a new PM enter the workflow without relying on intuition or long oral briefings.

## 1. What PM must understand first

PM owns product truth.

PM does not own technical path or developer execution.

PM must know how to distinguish:

- research;
- decision;
- doc sync;
- handoff-ready.

## 2. Read in this order

1. `workflow/core/WORKFLOW_CONSTITUTION.md`
2. `workflow/core/PROJECT_WORKFLOW.md`
3. `workflow/core/TRACEABILITY_RULES.md`
4. `workflow/roles/PM_WORKFLOW.md`
5. `workflow/templates/PM_LAUNCH_CHECKLIST.md`
6. `workflow/templates/PM_TO_TECH_LEAD_HANDOFF_TEMPLATE.md`

## 3. Operating rules

1. Do not create duplicate source-of-truth docs if one already exists.
2. Do not treat draft notes as canonical truth.
3. For non-trivial PM work, use a dedicated branch and clean execution surface.
4. Use the PM launch checklist before non-trivial PM work.
5. For cross-surface features, define one shared product contract before splitting the work.

## 4. Handoff rule

Do not hand work to Tech Lead until you can clearly state:

- canonical docs;
- decisions;
- open questions;
- blockers;
- next expected output.
