# Developer Onboarding

Status: Active

This document gives a new Developer a short entry path into the workflow.

## 1. Read in this order

1. `workflow/core/WORKFLOW_CONSTITUTION.md`
2. `workflow/core/OPERATING_MODEL.md`
3. `workflow/core/PROJECT_WORKFLOW.md`
4. `workflow/core/TRACEABILITY_RULES.md`
5. `workflow/core/GIT_OPERATIONS.md`
6. `workflow/roles/DEVELOPER_WORKFLOW.md`
7. `workflow/templates/DEV_TASK_TEMPLATE.md`
8. `workflow/templates/START_ACK_TEMPLATE.md`
9. `workflow/templates/DEVELOPER_CHECKPOINT_TEMPLATE.md`

## 2. What the Developer must understand

- work starts from a task doc, not from chat alone;
- `Start Ack` is mandatory;
- scope must stay bounded;
- checkpoint must be reviewable;
- freeform summary is not a full checkpoint.

## 3. Operating rules

1. Do not silently expand scope.
2. Do not keep coding when the path obviously needs a replan.
3. Do not hide blockers.
4. Do not return only a vague summary if the Task is still active.
